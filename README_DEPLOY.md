# Инструкция по запуску сайта в интернете с Админ-панелью

Чтобы сайт работал в интернете и вы могли управлять им через админ-панель (добавлять проекты, менять стили) с любого устройства, мы перевели систему на **Firebase** (Google Cloud).

## Шаг 1: Настройка Firebase (База данных и Вход)

1. Зайдите на сайт [Firebase Console](https://console.firebase.google.com/) (войдите через Google почту).
2. Нажмите **"Create a project"** (Создать проект), назовите его (например `oyu-design`), выключите Google Analytics (не обязательно) и создайте.
3. **Создание базы данных:**
   - В меню слева выберите **Build -> Firestore Database**.
   - Нажмите **Create Database**.
   - Выберите регион (можно оставить по умолчанию).
   - **ВАЖНО:** На этапе "Secure rules" выберите **Start in Test mode** (Тестовый режим). Это позволит сайту читать и писать данные.
     *(В будущем для безопасности можно будет настроить правила, но для начала Test mode подойдет).*

4. **Настройка входа (Auth):**
   - В меню слева выберите **Build -> Authentication**.
   - Нажмите **Get Started**.
   - Выберите **Email/Password** и включите переключатель **Enable**.
   - Перейдите во вкладку **Users** и нажмите **Add user**.
   - **Email:** `admin@oyudesign.group` (Именно этот! Я настроил код на него).
   - **Password:** Придумайте надежный пароль. Этот пароль вы будете вводить на сайте.

## Шаг 2: Получение ключей

1. В настройках проекта (шестеренка рядом с Project Overview -> Project settings).
2. Прокрутите вниз до раздела **Your apps**.
3. Нажмите на иконку **Web** `</>`.
4. Введите название (например `OyuWebsite`) и нажмите Register.
5. Вы увидите код `const firebaseConfig = { ... };`.
6. Скопируйте всё содержимое внутри фигурных скобок `{ ... }`.

## Шаг 3: Вставка ключей в код

1. Откройте файл `firebase-app.js` на вашем компьютере.
2. Замените строки `YOUR_API_KEY` и другие на значения, которые вы скопировали из Firebase.

Пример, как должно выглядеть:
```javascript
const firebaseConfig = {
    apiKey: "AIzaSyD...",
    authDomain: "oyu-design.firebaseapp.com",
    projectId: "oyu-design",
    storageBucket: "...",
    messagingSenderId: "...",
    appId: "..."
};
```

## Шаг 4: Публикация на GitHub (Единственный способ)

1. **Создайте репозиторий:**
   - Зайдите на [github.com](https://github.com/) и зарегистрируйтесь (если нет аккаунта).
   - Нажмите значок "плюс" (+) в правом верхнем углу → **New repository**.
   - Название (Repository name): `oyu-design`.
   - Выберите: **Public**.
   - Нажмите: **Create repository**.

2. **Загрузите файлы:**
   - На открывшейся странице найдите ссылку: *Create a new file or upload an existing file*. Нажмите **uploading an existing file**.
   - Перетащите **ВСЮ вашу папку с сайтом** (все файлы: index.html, admin.html, script.js, logo, styles и т.д.) прямо в окно браузера.
   - Дождитесь загрузки и внизу нажмите зеленую кнопку **Commit changes**.

3. **Включите сайт:**
   - В вашем репозитории перейдите в **Settings** (Настройки сверху).
   - В меню слева найдите раздел **Pages**.
   - Под надписью **Build and deployment** -> **Branch** выберите `main` (или `master`) и нажмите **Save**.

4. **Готово!**
   - Через 1–2 минуты сверху обновите страницу. Там появится ссылка на ваш сайт (например: `https://vash-login.github.io/oyu-design/`).

## Шаг 5: Безопасность и Админ-панель (Ответ на ваш вопрос)

Вы спросили: *"Где создать логин/пароль и как защитить админку?"*

**1. Где создается логин и пароль?**
Вы создаете их сами в **Firebase Console** (это было в Шаге 1). 
- Логин (Email): `admin@oyudesign.group`
- Пароль: Вы придумали его сами, когда нажимали **Add user** в разделе **Authentication**.
- **Как изменить пароль?** Зайдите снова в Firebase -> Authentication -> Наведите на пользователя -> Нажмите карандаш или "Reset Password".

**2. Защищена ли админ-панель?**
- **Да.** Страница `admin.html` доступна в интернете, НО никто не сможет ничего сделать, пока не введет правильный логин и пароль.
- Без входа кнопки "Добавить", "Удалить" и "Редактировать" просто не сработают (Firebase отклонит запрос).
- Даже если кто-то узнает адрес админки, он увидит только экран входа. Данные защищены на серверах Google.
- **ВАЖНО:** Для полной защиты обязательно выполните этот шаг в Firebase:
   1. Зайдите в **Firestore Database** -> **Rules**.
   2. Замените код на этот:
   ```javascript
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read: if true;
         allow write: if request.auth != null;
       }
     }
   }
   ```
   3. Нажмите **Publish**. Теперь хакеры точно не пройдут!

**3. Можно ли скрыть саму страницу admin.html?** (Дополнительно)
- Технически страница есть, но вы можете не давать на нее ссылку в меню. Она будет "секретной" (известной только вам). Для этого просто удалите ссылку `<a href="admin.html">` из файлов `index.html` и `projects.html`.

## Как пользоваться сайтом

1. Откройте вашу ссылку (которую дал GitHub).
2. Чтобы управлять сайтом, допишите `/admin.html` в конце адреса (или нажмите на скрытую ссылку).
3. Введите ваш email (`admin@oyudesign.group`) и пароль.
4. Управляйте проектами! Все изменения сразу увидят посетители.
