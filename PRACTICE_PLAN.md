# Отчет о выполнении технического задания (ТЗ)
## Проект: Сайт портфолио для студии дизайна "OYU Design Group"
**Исполнитель:** Омурбек
**Срок выполнения:** 19 дней

---

### План-график выполнения работ

#### **Этап 1: Подготовка и Дизайн (Дни 1-4)**
*   **День 1:**
    *   Анализ технического задания.
    *   Подбор референсов и цветовой палитры (темно-зеленый Main, бордовый Accent).
    *   Настройка рабочего окружения, создание структуры проекта (HTML, CSS).
*   **День 2:**
    *   Верстка шапки сайта (Header) и главной секции (Hero).
    *   Подбор и подключение шрифтов (`SF Pro Display`).
*   **День 3:**
    *   Верстка секции "О студии".
    *   Работа с изображениями, настройка CSS фильтров и наложений.
*   **День 4:**
    *   Верстка секции "Портфолио" (CSS Grid).
    *   Создание карточек проектов с эффектами при наведении (Hover).

#### **Этап 2: Основной функционал и Анимации (Дни 5-9)**
*   **День 5:**
    *   Верстка формы обратной связи и "Подвала" (Footer).
    *   Верстка страницы `projects.html`.
*   **День 6:**
    *   Написание JavaScript для мобильного меню (Бургер).
    *   Реализация плавной прокрутки по якорям.
*   **День 7:**
    *   Подключение библиотеки анимаций GSAP.
    *   Настройка анимаций появления элементов при скролле.
*   **День 8:**
    *   Создание модальных окон для просмотра деталей проектов.
    *   Верстка `admin.html` (форма входа и панель управления).
*   **День 9:**
    *   Написание логики открытия/закрытия модальных окон.
    *   Тестирование переходов между страницами.

#### **Этап 3: Разработка Админ-панели (Дни 10-14)**
*   **День 10:**
    *   Реализация логики авторизации (первичная версия на LocalStorage).
    *   Создание функций для добавления новых проектов (Create).
*   **День 11:**
    *   Реализация отображения списка проектов в админке (Read).
    *   Функции удаления проектов (Delete).
*   **День 12:**
    *   Реализация функции редактирования существующих проектов (Update).
    *   Сохранение изменений в браузере.
*   **День 13:**
    *   Изучение документации Firebase (Google Cloud).
    *   Регистрация проекта в Firebase Console.
*   **День 14:**
    *   Подключение Firebase SDK к проекту.
    *   Настройка файлов конфигурации (`firebase-app.js`).

#### **Этап 4: Backend и База данных (Дни 15-17)**
*   **День 15:**
    *   Миграция данных с LocalStorage на Cloud Firestore.
    *   Настройка асинхронного получения данных (`async/await`).
*   **День 16:**
    *   Настройка Firebase Authentication (Email + Password).
    *   Настройка правил безопасности базы данных (Firestore Rules).
*   **День 17:**
    *   Обработка ошибок сети и загрузки.
    *   Добавление индикаторов загрузки (Loaders) и заглушек для пустых списков.

#### **Этап 5: Адаптация, Тестирование и Деплой (Дни 18-19)**
*   **День 18:**
    *   **Мобильная адаптация:**
        *   Исправление отображения изображений в блоке "О студии" (проблема обрезки).
        *   Настройка вертикального отображения иконок в footer.
        *   Адаптация карточек проектов под мобильные экраны.
    *   Создание страницы `404.html` для обработки ошибок.
*   **День 19:**
    *   **Финальная полировка UX/UI:**
        *   Реализация бесшовных переходов между секциями ("Туманный эффект").
        *   Интеграция с Gmail для мгновенного написания писем.
        *   Микро-анимации навигации (увеличение и приподнимание текста).
        *   Оптимизация мобильных модальных окон (фиксация кнопки заказа).
    *   Публикация проекта на GitHub Pages.
    *   Финальное тестирование на разных устройствах.
    *   Написание итогового отчета.

---

### Итог
За 19 дней был разработан полностью функциональный веб-ресурс с собственной системой управления контентом, защищенной базой данных и адаптивным дизайном. Проект готов к сдаче.
